# Набор ИИ-скиллов для SS13/BYOND — руководство

Это человеческое описание четырёх скиллов из этого репозитория. Скилл — это пакет инструкций, который ИИ-агент (Claude и совместимые) подгружает, когда берётся за подходящую задачу. Скиллы не содержат привязок к конкретной машине: их можно установить у себя и работать с любым SS13-форком.

## Из чего состоит набор

| Скилл | Одной фразой |
|---|---|
| **byond-codemaster-controller** | Диспетчер: решает, какой скилл дёргать, и управляет «семантическими базами» репозиториев |
| **byond-ss13-coding** | Универсальный кодинг на Dream Maker: язык, архитектура SS13, производительность, ревью |
| **ss13-tgui** | Всё про TGUI и браузерные интерфейсы: React-фронтенд, DM-бэкенд, отладка белых окон |
| **tgstation-modular-content** | Как добавлять контент на форк так, чтобы он переживал синхронизации с upstream |

Принцип экономии: агент подгружает только тот скилл (и только те его справочные файлы), которые нужны текущей задаче. Ничего «на всякий случай».

## byond-codemaster-controller — диспетчер

«Скилл над скиллами». С него начинается любая нетривиальная задача: он определяет, какая работа предстоит (чистый DM? интерфейс? модульный контент? комбинация?), и подключает нужные скиллы в правильном порядке. Второй его домен — семантические базы (`ai_navigation/`): справочные слои по конкретному репозиторию, которые он умеет находить, проверять на свежесть и строить с нуля для незнакомой кодовой базы.

Примеры запросов, на которые он срабатывает:

- EN: "Set up a navigation base for this new fork and tell me which skills apply to my task."
- RU: «Собери семантическую базу для нового форка и скажи, какие скиллы нужны под мою задачу».

## byond-ss13-coding — универсальный DM/SS13 кодинг

Основной рабочий скилл. Знает семантику языка Dream Maker и движка BYOND (проверенную по официальному справочнику), архитектуру SS13 (Master Controller, подсистемы, компоненты, сигналы, жизненный цикл объектов и qdel), правила производительности (бюджет тика, аллокации, appearances) и процедуру код-ревью с чеклистом. Отличает «истину движка» от «конвенций проекта»: не навязывает стиль tgstation форку с другой архитектурой.

Примеры запросов:

- EN: "Add a component that regenerates the wearer every 5 seconds and clean it up properly on deletion."
- RU: «Добавь компонент, который регенерирует носителя каждые 5 секунд, и корректно очисти его при удалении».
- EN: "Review this DM patch for hard-delete risks."
- RU: «Проверь этот DM-патч на риски hard delete».

## ss13-tgui — интерфейсы

Специализированный скилл для всего, что связано с TGUI: жизненный цикл окна, `ui_data`/`ui_act`, React-компоненты и стили, мост между BYOND и браузером, доставка ассетов, диагностика пустых/белых окон на разных платформах. Кодинг-скилл сам передаёт задачу сюда, как только она пересекает границу «DM-данные → интерфейс».

Примеры запросов:

- EN: "My tgui window opens blank on Windows 7 clients — help me triage it."
- RU: «TGUI-окно открывается белым у клиентов на Windows 7 — помоги разобраться».

## tgstation-modular-content — модульный контент

Скилл о том, *куда класть* изменения на форке, который регулярно подтягивает upstream: модульные папки, оверлеи, теги правок, агрегаторные включения. Его цель — чтобы ваши фичи не превращались в конфликты при каждом ребейзе. Работает в паре с кодинг-скиллом: этот решает «куда», тот — «как».

Примеры запросов:

- EN: "Add a new job item on our fork without touching upstream files."
- RU: «Добавь новый предмет профессии на нашем форке, не трогая upstream-файлы».

## Как они работают вместе

1. Контроллер смотрит на задачу и раздаёт роли (таблица диспетчеризации в его SKILL.md).
2. Для комбинированных задач порядок фиксирован: **размещение важнее реализации** (modular-content решает структуру файлов, кодинг/tgui — содержимое) и **DM-сторона отделена от фронтенда** (byond-ss13-coding ведёт до границы `ui_data`/`ui_act`, дальше ss13-tgui).
3. Иерархия истины: живой код репозитория > семантическая база (факты форка) > скилл (общие закономерности). На семантике движка последнее слово — за byond-ss13-coding.

## Установка

1. Возьмите папку скилла (или упакованный `.skill` — это обычный zip) и добавьте её в хранилище скиллов вашего агента (в приложении Claude — через панель Skills → Add).
2. Скиллы самодостаточны: никаких путей настраивать не нужно. Расположение репозиториев и баз агент выясняет сам по протоколу обнаружения.

## Честность

Скиллы прямо запрещают агенту утверждать, что код «скомпилирован» или «проверен», если соответствующие команды реально не запускались. Вместо этого агент обязан перечислить, какие проверки (компиляция, DreamChecker, CI) можете запустить вы.
